# 05-01 Plan: Pilot Harness and Artifact Validator

<objective>
Create the Phase 5 pilot execution harness and persisted-artifact validator so the four main behavioral methods can run on three small synthetic seeds with durable REP-03 status/provenance rows, utility logs, and metric sanity checks.
</objective>

## Scope

In scope:

- Add a Phase 5 wrapper that filters the main behavioral methods explicitly.
- Run only synthetic pilot scenarios at the selected small scale and seed list.
- Write runner-compatible raw, metric, and utility artifacts under `results/pilot/phase05/`.
- Validate persisted CSV artifacts for required fields, statuses, metric ranges, and utility-log joinability.
- Add focused tests for the wrapper and validator.

Out of scope:

- Matched-coverage and fixed accepted-set diagnostics, owned by `05-02`.
- Final pilot report and bug ledger closeout, owned by `05-03`.
- Beijing-inspired or formal paired-seed evidence.
