# Plan 06-04: Robustness and Algorithm Diagnostics

<objective>
Run the remaining roadmap-listed supplementary packages: meeting-point density and walking tolerance, fleet/demand stress tests, rolling-horizon diagnostics, equity analysis, and ALNS/MILP algorithm validation, with independent outputs and gates.
</objective>

<execution_addendum>
Current 06-04 execution also includes the user-requested formal utility-sensitivity diagnostics covering beta_walk, beta_wait/access-time, beta_ivt, service ASC, outside option, and passenger type shares. Canonical 06-04 outputs are written under `results/formal/phase06/robustness/`; `experiments/phase06_supplementary.py` remains a compatibility CLI that routes the original supplementary package names into the robustness runner.
</execution_addendum>

<must_haves>
- D-07: Phase 6 should complete meeting-point density, fleet/demand, rolling-horizon, equity, and ALNS/MILP supplementary packages.
- D-08: Supplementary packages are reported as main-text robustness summaries plus appendix/supplement detail.
- D-13: Failed supplementary packages cannot support their corresponding robustness claim.
- D-16: Coverage and efficiency metrics must be reported together.
- D-18: Claim gating requires no unexplained critical supplementary conflict.
</must_haves>

<truths>
- `analysis/sensitivity.py` has walking-radius and fleet-size patterns but writes legacy result files and must be routed through formal output paths.
- `analysis/equity.py` computes type-level acceptance/wait/walk/IVT but uses only 3 seeds and one scale by default.
- `experiments/algorithm_diagnostics.py` provides ALNS budget smoke diagnostics.
- `experiments/milp_gap.py` provides static fixed accepted-set MILP diagnostics and no-Gurobi rows.
- Current `FullModel` rolling-horizon metadata must be reconciled before rolling-horizon diagnostics are interpreted.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Robustness package writes to legacy result paths | medium | Force package outputs under `results/formal/phase06/supplementary/` |
| Equity results are overclaimed as real population evidence | medium | Label passenger types as simulation-range constructs and report limitations |
| Algorithm diagnostic is treated as service-design evidence | high | Persist `evidence_family = algorithm_diagnostic` and diagnostic roles |
| MILP absence blocks unrelated evidence | low | Record `no_gurobi` as durable non-comparable diagnostic unless a specific claim depends on it |
</threat_model>
