# Plan 06-02: Main Behavioral Formal Matrix

<objective>
Run, validate, and close the required 20-seed main behavioral synthetic matrix for the four approved service-design methods across the formal scale grid, preserving all raw rows, utility logs, configs, failures, timeouts, and reruns.
</objective>

<must_haves>
- D-01: Minimum formal completion target is 20 paired synthetic seeds.
- D-02: Main evidence table includes only the four core behavioral service-design methods.
- D-04: Main scales are exactly `100`, `200`, `300`, and `500`.
- D-05: Optional seeds 21-30 upgrade the evidence base only if all extra paired cells complete.
- D-11: Failed or timeout rows remain durable raw rows and reruns use the same `seed x scale x method` cell.
- D-12: Any unresolved failed or timeout row in the main matrix blocks main evidence passage.
- D-16: Main metrics must include `total_vehicle_km`, `vkm_per_served_trip`, `vkm_per_original_request`, and `served_share`.
</must_haves>

<truths>
- Plan 06-01 owns the runner wrapper and validator.
- The Phase 6 main matrix has 320 required cells: 20 seeds x 4 scales x 4 methods.
- Phase 5 pilot results are readiness-only and cannot be cited as formal evidence.
- The main matrix must not include deterministic or algorithm diagnostic variants.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Long formal runs stall without closure | high | Run by manifest-defined cells, validate persisted artifacts, and maintain a rerun ledger |
| Partial optional seed extension is misreported as 30 seeds | high | Upgrade only if every seed 21-30 paired cell completes |
| Failed cells are accidentally dropped from processed tables | high | Raw failed/timeout rows remain in CSV and ledger before any processing |
| Formal command overwrites unrelated results | medium | Use `results/formal/phase06/main/` only |
</threat_model>
