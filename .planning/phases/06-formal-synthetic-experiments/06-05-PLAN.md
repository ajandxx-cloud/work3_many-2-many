# Plan 06-05: Statistical Synthesis and Evidence Report

<objective>
Generate paired-difference tables, paired bootstrap confidence intervals, processed formal result tables, plots, and the Phase 6 evidence report after the main matrix and supplementary package gates have completed.
</objective>

<must_haves>
- D-08: Supplementary packages are reported as main-text summaries plus appendix/supplement detail.
- D-09: Critical supplementary conflicts are explained, rerun/corrected, or carried forward as claim limitations.
- D-14: Failure/rerun ledger is summarized in the formal report.
- D-15: Main tables prioritize paired differences against baselines within the same seed.
- D-16: Main table reports `total_vehicle_km`, `vkm_per_served_trip`, `vkm_per_original_request`, and `served_share`.
- D-17: Paired bootstrap 95% confidence intervals are the primary uncertainty summary.
- D-18: Results can enter Phase 8 only when matrix, quartet metrics, CIs, and critical supplementary conflicts are closed.
</must_haves>

<truths>
- Phase 2 statistical plan specifies paired differences and confidence intervals.
- Phase 6 does not approve final manuscript claims; Phase 8 owns claim-evidence grading.
- Failed, timeout, no-Gurobi, empty-set, and conflict rows must remain visible in reports.
- Tables and plots must not hide coverage, acceptance, or rejection-rate differences.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Processed tables drop incomplete pairs | high | Pairing logic must emit `n_valid_pairs` and missing-pair diagnostics |
| CI method is opaque or irreproducible | medium | Persist bootstrap seed, resample count, input path, and command |
| Phase 6 report approves final claims | high | Report evidence only and route claim grading to Phase 8 |
| Plots imply unsupported win/loss claims | medium | Include coverage/rejection context and limitations |
</threat_model>
