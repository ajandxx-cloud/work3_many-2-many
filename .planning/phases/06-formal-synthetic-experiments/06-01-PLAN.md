# Plan 06-01: Formal Harness, Manifests, and Validators

<objective>
Create the Phase 6 formal experiment harness and validators that predeclare the 20-seed main matrix, isolate formal outputs, filter exactly the approved behavioral-main methods, preserve provenance, and block invalid result matrices before any paper-facing tables are generated.
</objective>

<must_haves>
- D-01: Main formal evidence uses at least 20 paired synthetic seeds and only upgrades to 30 if every optional paired cell completes.
- D-02: Main evidence includes only `DoorToDoor_Choice_CommonRouting`, `SingleSidedPickup_Choice_CommonRouting`, `SingleSidedDropoff_Choice_CommonRouting`, and `BidirectionalMP_Choice_RH_ALNS`.
- D-03: Diagnostic methods must not appear as main evidence methods.
- D-04: Formal main scales are exactly `100`, `200`, `300`, and `500`; scale `20` is pilot-only.
- D-11: Failed and timeout rows remain durable and are never silently deleted.
- D-14: The run-level ledger records `run_id`, `config_id`, `seed`, `scale`, `method`, `status`, `error`, `reason`, `fix`, and `rerun_result`.
</must_haves>

<truths>
- `experiments/runner.py` already writes durable raw rows, metrics rows, utility logs, provenance fields, and error rows.
- `experiments/phase05_pilot.py` already demonstrates a safe variant-filtering wrapper around the shared runner.
- `experiments/pilot_validation.py` already checks persisted schema, matrix completeness, statuses, numeric ranges, and utility joinability.
- The current `FullModel` implementation label must be validated before formal main runs because label/implementation drift would invalidate headline evidence.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Formal outputs overwrite legacy result files | medium | Write only under `results/formal/phase06/` by default |
| Diagnostics leak into the main table | high | Filter by exact method labels and `evidence_family == behavioral_main` |
| Replacement seeds hide failed cells | high | Predeclare required and optional seeds in `formal_seed_manifest.json` |
| Method metadata overclaims rolling-horizon/ALNS behavior | high | Add a blocking label/implementation validation check before main matrix execution |
| Local output paths delete unrelated user artifacts | medium | Create subdirectories; never recursively delete outside `results/formal/phase06/` |
</threat_model>
