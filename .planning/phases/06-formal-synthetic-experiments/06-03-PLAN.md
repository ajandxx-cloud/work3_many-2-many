# Plan 06-03: Critical Supplementary Controls

<objective>
Implement, run, and gate the critical supplementary packages that directly affect interpretation of the main efficiency claim: matched coverage, fixed accepted-set routing, and utility sensitivity.
</objective>

<must_haves>
- D-06: Phase 6 uses a main evidence plus independent supplementary packages structure.
- D-07: Matched coverage, fixed accepted-set, and utility sensitivity packages must complete or produce explicit critical conflicts.
- D-09: A critical supplementary conflict blocks direct advancement to Phase 8 until explained, rerun, corrected, or downgraded.
- D-10: Matched coverage, fixed accepted-set, and utility sensitivity are critical for headline behavioral claims unless narrowed by documented rationale.
- D-13: Supplementary package failures are handled through independent package gates.
- D-18: Results enter Phase 8 only when no critical supplementary conflict remains unexplained.
</must_haves>

<truths>
- Phase 5 closed matched-coverage pilot blockers through per-seed integer target counts.
- Phase 5 fixed accepted-set smoke used `common_candidate_serviceable` as a diagnostic fallback only.
- Phase 2 defines matched coverage and fixed accepted-set diagnostics as supplementary controls, not substitutes for main behavioral evidence.
- Phase 3 defines one-at-a-time sensitivity emphasis around service ASC, outside option, type shares, and walk sensitivity.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Matched coverage silently adjusts targets | high | Persist target basis, target count, adjusted flag, achieved count, and tolerance |
| Fixed accepted-set fallback is overinterpreted | high | Persist construction rule and label candidate-serviceable fallback as diagnostic only |
| Sensitivity outputs reuse old ambiguous metric labels | medium | Use formal quartet metric names and explicit parameter manifests |
| Supplementary failure is ignored | high | Write package gate rows with critical/non-critical status and block critical unresolved conflicts |
</threat_model>
