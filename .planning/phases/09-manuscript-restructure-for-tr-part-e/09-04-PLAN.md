# Plan 09-04: Experiment Section and Evidence-Family Plan

<objective>
Create the manuscript experiment-section plan that separates experimental design,
formal main evidence, robustness controls, equity evidence, and algorithm
diagnostics, using Phase 2/6 metric and evidence-family contracts as the
paper-facing structure.
</objective>

<must_haves>
- D-07: Experiment section is design first, then evidence packages.
- D-10: Main table contains only formal main evidence matrix with approved metrics and paired differences.
- D-11: Matched-coverage and fixed accepted-set controls are robustness summaries, with details in appendix/supplement.
- D-12: Algorithm diagnostics are scoped and mostly appendix/supplement detail.
- D-13: Equity appears as a main-text trade-off summary with details in appendix/supplement.
- D-14: No ambiguous `vkm_per_trip` language.
- D-17: Beijing/case language remains synthetic unless later Phase 7 evidence exists.
</must_haves>

<truths>
- Current `experiments.tex` mixes main results, matched-coverage diagnostics, MILP/ALNS diagnostics, Beijing-inspired results, sensitivity, equity, gamma, and weight sensitivity.
- Phase 2 defines the metric dictionary and forbids denominator ambiguity.
- Phase 6 formal outputs are not present in this workspace, so final result values must remain placeholders.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Experiment section reports old exploratory numbers as formal evidence | high | Use placeholders and require Phase 6/8 artifacts before final values |
| Diagnostic rows leak into main evidence table | high | Define evidence-family table and section allocation |
| Metrics revert to `vkm_per_trip` | high | Add forbidden-metric language and required quartet checks |
| Beijing-inspired scenario is overstated | medium | Require exact synthetic boundary wording |
</threat_model>
