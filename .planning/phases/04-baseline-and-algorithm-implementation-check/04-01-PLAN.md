# 04-01 Plan: Shared Behavioral Baselines and Method Mapping

<objective>
Make the four behavioral service-design baselines runnable under one actual-offer choice harness, add the missing SingleSidedDropoff service design, and document the concept-label mapping so Phase 5 pilots cannot mix legacy variant names or deterministic diagnostics into the behavioral evidence family.
</objective>

## Scope

In scope:

- Add or validate DoorToDoor + Choice, SingleSidedPickup + Choice, SingleSidedDropoff + Choice, and BidirectionalMP + Choice.
- Preserve the Phase 3 actual-offer choice path and explicitly guard against `_mnl_filter_requests`.
- Add concept-level method metadata for service design, choice model, reoptimization, routing solver, evidence family, diagnostic role, and legacy class.
- Create `VARIANT_MAPPING.md` and seed `04_IMPLEMENTATION_AUDIT.md` with the baseline implementation status.

Out of scope:

- Formal pilot or paired-seed experiment execution.
- Large-scale ALNS runtime-quality benchmarking.
- Full class renames that increase implementation risk.
- MILP scope validation, which is owned by `04-04`.
