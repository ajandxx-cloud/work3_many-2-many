# 04-04 Plan: MILP Diagnostic Scope and Exact-Snapshot Validation

<objective>
Validate and document the limited MILP/exact diagnostic path so Phase 4 can compare ALNS/greedy against exact-style evidence only when semantics match, while preserving a no-Gurobi fallback and avoiding claims that the current MILP is a full online DRT benchmark.
</objective>

## Scope

In scope:

- Document the exact boundaries of `src/drt/milp.py` and `experiments/milp_gap.py`.
- Add pure-Python tiny-instance fixture checks that run without Gurobi.
- Keep Gurobi-backed solve tests optional and skipped when solver/license is unavailable.
- Align MILP gap outputs with the runner/schema and evidence-family language.
- Update `04_ALGORITHM_VALIDATION.md` and `04_IMPLEMENTATION_AUDIT.md`.

Out of scope:

- Implementing a full route-sequencing exact online DRT model.
- Running large MILP experiments.
- Treating simplified MILP objective values as formal service-design evidence.
- Manuscript claim approval.
