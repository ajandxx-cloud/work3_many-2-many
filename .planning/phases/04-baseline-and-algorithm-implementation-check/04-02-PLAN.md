# 04-02 Plan: Unified Runner Schema, Failure Rows, and Baseline Validation

<objective>
Make runner outputs durable and schema-consistent across behavioral and diagnostic methods, including bounded timeout handling, concept-label method metadata, formal metric names, and validation documentation before Phase 5 pilot runs.
</objective>

## Scope

In scope:

- Add run/provenance/result schema fields to raw runner rows.
- Persist failed, timeout, and infeasible runs as durable rows with status and error message.
- Replace or supplement ambiguous vehicle-km-per-trip outputs with `vkm_per_served_trip` and `vkm_per_original_request`.
- Fix the known thread-pool timeout behavior so timeout tests do not wait for the worker to finish.
- Create `04_BASELINE_VALIDATION.md`.

Out of scope:

- Running Phase 5 pilot seeds.
- Changing manuscript tables or final claims.
- Adding ALNS operator diagnostics, which is owned by `04-03`.
- Changing MILP formulation scope, which is owned by `04-04`.
