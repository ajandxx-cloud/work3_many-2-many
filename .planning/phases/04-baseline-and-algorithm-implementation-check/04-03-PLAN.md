# 04-03 Plan: Greedy, No-Rolling-Horizon, and ALNS Diagnostics

<objective>
Expose and validate algorithm diagnostics for greedy insertion, no-rolling-horizon behavior, and ALNS convergence/operator health so Phase 4 can distinguish algorithm credibility checks from behavioral service-design evidence.
</objective>

## Scope

In scope:

- Add a named greedy insertion diagnostic baseline.
- Validate no-rolling-horizon as an algorithm diagnostic under shared choice semantics.
- Add ALNS trace collection for objective, runtime, accepted/unassigned counts, and operator contribution.
- Add a small multi-budget diagnostic path for 5/20/50 iterations or comparable local values.
- Create or update `04_ALGORITHM_VALIDATION.md` with algorithm diagnostic results and boundaries.

Out of scope:

- Full large-scale runtime-quality benchmarking.
- Formal paired-seed evidence or pilot execution.
- Exact/MILP scope validation, which is owned by `04-04`.
- Manuscript claim approval.
