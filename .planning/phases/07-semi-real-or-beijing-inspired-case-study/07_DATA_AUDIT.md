# Phase 7 Data Audit

**Phase:** 07 - Semi-Real or Beijing-Inspired Case Study
**Date:** 2026-06-16
**Status:** complete
**Classification:** Beijing-inspired synthetic case

## Audit Question

Does the current repository contain real or semi-real case-study data that can
support a Phase 7 real or semi-real case study?

## Decision

The current repository does not contain real or semi-real OD demand, road
network, public transit stop, POI, or operational DRT data. The available
Beijing-labeled material is a Beijing-inspired synthetic scenario generated by
code. It must not be described as a real Beijing case, a semi-real Beijing case,
or direct evidence for Beijing operations.

## Evidence Reviewed

| Area | Evidence | Audit result |
|---|---|---|
| Data directories | No active `data/`, `datasets/`, or imported case-study dataset directory exists in the repository. Active directories are `analysis/`, `docs/`, `experiments/`, `manuscript/`, `results/`, `src/`, and `tests/`. | No local real/semi-real data store found. |
| Scenario generator | `experiments/scenarios.py` defines `generate_synthetic()` and `generate_beijing()`. | Both are code-generated simulation scenarios. |
| Beijing scenario OD | `generate_beijing()` draws origins uniformly on a 15 km x 15 km grid and destinations from a Gaussian offset clipped to the grid. | Synthetic OD, not observed OD. |
| Beijing scenario network | Coordinates are Euclidean grid coordinates; there is no road-network graph, travel-time matrix, GTFS network, or map-matching artifact. | Synthetic Euclidean grid, not real road network. |
| Meeting points | `generate_beijing()` places 80 meeting points from a 9 x 9 regular grid at 1875 m spacing. | Artificial meeting-point grid, not POI, transit stop, or road-node data. |
| Request times | `generate_beijing()` samples request times from a simulated morning-peak distribution centered around 8 am and clipped to 7-9 am. | Simulated time distribution. |
| Passenger preferences | `experiments/config.py` states the Phase 3 choice values are inherited/simulation-range inputs and are not real-data calibration. | Simulation-range passenger types and preferences. |
| Fleet setting | `experiments/config.py` fixes `BEIJING_SCALE = 200`; runner uses 15 vehicles for that scale. | Experimental setting, not observed fleet operations. |
| Existing Beijing rows | `results/beijing_results.csv` has 21 legacy rows: 3 seeds x 7 legacy variants. | Existing synthetic illustrative output only; no Phase 7 formal validation package. |
| Public-data note | `docs/工作3公开数据集.txt` is a recommendation note for possible external datasets such as taxi OD and GTFS. | Suggestions only; no imported data assets. |
| Manuscript/docs language | Current manuscript and legacy result prose include synthetic caveats but also contain risky policy/deployment wording, such as policy recommendations for Chinese city operators and references to real deployments. | Overclaim risk remains; Phase 7 must bound wording. |

## Classification

**Beijing-inspired synthetic case**

This is not a real case. This is not a semi-real case. The repository has
insufficient data for a real or semi-real case study, but it does contain a
bounded Beijing-inspired synthetic scenario and legacy illustrative outputs.

## Consequence For Phase 7

Phase 7 should close through bounded synthetic-case documentation rather than a
new real/semi-real experiment. Any case-study wording must state the synthetic
nature of the scenario before discussing results.
