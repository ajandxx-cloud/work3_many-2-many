# 03-03 Plan: Variant Integration, Status Schema, and Utility Logging

<objective>
Integrate the Phase 3 choice evaluator into the experiment layer so behavioral variants evaluate actual feasible offers, emit durable request statuses, preserve utility-component logs, and expose tests that prevent proxy MNL filtering from returning.
</objective>

## Scope

In scope:

- Replace proxy `_mnl_filter_requests()` behavior in behavioral paths with actual-offer acceptance after feasible insertion/offer construction.
- Add row-level status and detailed reason fields needed by Phase 2 metrics.
- Add separate utility-component logging artifacts linked to raw passenger rows.
- Add configuration hooks for choice parameters, sensitivity values, and seeded passenger type shares.
- Add tests proving shared response semantics, deterministic type assignment, status separation, and log schema.

Out of scope:

- Implementing `SingleSidedDropoff` as a full Phase 4 baseline unless it is a small prerequisite for shared-response tests.
- Running pilot or formal experiments.
- Redesigning ALNS/MILP diagnostics.
- Editing manuscript sections or interpreting new results.
