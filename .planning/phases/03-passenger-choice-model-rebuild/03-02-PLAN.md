# 03-02 Plan: Core Choice Evaluator, Passenger Types, and Utility Components

<objective>
Replace the hidden-normalized proxy choice helper with reusable core choice primitives that evaluate one actual feasible offer, expose service ASC and outside-option utility, deterministically assign passenger types, and return utility components suitable for row-level logging.
</objective>

## Scope

In scope:

- Extend or wrap the existing `Bundle`/choice API so a concrete offered service bundle carries all utility-relevant attributes.
- Add a parameter/config object for service ASC, outside option constant, coefficient values, type shares, and RNG behavior.
- Add deterministic request-level passenger type assignment.
- Add a utility component result object that can be logged by experiment variants.
- Add focused tests for probabilities, outside option sensitivity, service ASC sensitivity, type assignment determinism, and no proxy utility on infeasible rows.

Out of scope:

- Rewriting the ALNS or insertion algorithm.
- Adding the missing `SingleSidedDropoff` service design.
- Running formal sensitivity experiments.
- Changing manuscript text or generated result files.
