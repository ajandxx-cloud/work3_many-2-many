# 03-01 Plan: Choice Contract, Calibration Table, and Sensitivity Design

<objective>
Create the Phase 3 passenger-choice contract and calibration document before code changes. These documents define the actual-offer single-choice semantics, outside option, service ASC, passenger types, seeded type shares, sensitivity grid, and output/logging obligations that the implementation plans must follow.
</objective>

## Scope

In scope:

- Define a single-offer acceptance model evaluated after a feasible service bundle exists.
- Define terminal status semantics for `choice_rejected` and `feasibility_rejected`.
- Define service ASC, outside-option constant, passenger type coefficients, type shares, and source labels.
- Define the sensitivity grid required before pilot and formal experiments.
- State how this contract supports the Phase 2 shared-response behavioral comparison.

Out of scope:

- Editing Python source code.
- Running pilot, formal, sensitivity, or manuscript experiments.
- Implementing fallback offers, multi-offer choice sets, or real-data calibration claims.
