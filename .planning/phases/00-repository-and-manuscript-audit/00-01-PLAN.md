# 00-01 Plan: Complete Repository and Manuscript Audit

**Phase:** 0 - Repository and Manuscript Audit
**Status:** planned
**Created:** 2026-06-15

## Objective

Complete Phase 0 by verifying the initialized audit artifacts against current code, manuscript sections, and result files. The output should let later phases proceed without guessing which claims, experiments, and result artifacts are trustworthy.

## Scope

In scope:

- Verify current repository map against source files and codebase-map artifacts.
- Trace manuscript tables, figures, and headline numbers to result files and scripts.
- Classify manuscript claims by evidence strength.
- Record gaps that block Phase 1+ planning.
- Update `.planning/STATE.md`.

Out of scope:

- Running new experiments.
- Modifying algorithms, metrics, or manuscript prose.
- Repairing git history or deleting stale tracked artifacts.
- Literature fact-checking beyond listing required targets for Phase 1.

## Tasks

1. **Result provenance audit**
   - Match manuscript headline numbers to `results/*.csv`, `results/*.json`, and figure scripts.
   - Mark each number as verified, ambiguous, or not reproducible.
   - Update `00_CURRENT_EXPERIMENT_MAP.md`.

2. **Claim provenance audit**
   - Scan abstract, experiments, policy, and conclusion sections.
   - Record each strong claim and classify current evidence.
   - Update `00_MANUSCRIPT_CLAIM_AUDIT.md`.

3. **Repository audit refinement**
   - Cross-check current code modules against `.planning/codebase/ARCHITECTURE.md` and `.planning/codebase/CONCERNS.md`.
   - Identify only Phase 0-relevant blockers.
   - Update `00_REPOSITORY_AUDIT.md`.

4. **Phase 0 gate decision**
   - Decide whether Phase 0 passes, passes with caveats, or remains blocked.
   - If passed, write `00-01-SUMMARY.md`.
   - Update `.planning/STATE.md` with next action.

## Verification

- No commands that generate new experiment results are run.
- Every audit claim references a concrete file path or explicitly states it is inferred from the review note/rebuild brief.
- Phase 0 summary lists remaining risks that should be handled in Phase 1 and Phase 2.

## Deliverables

- Updated `00_REPOSITORY_AUDIT.md`
- Updated `00_MANUSCRIPT_CLAIM_AUDIT.md`
- Updated `00_CURRENT_EXPERIMENT_MAP.md`
- `00-01-SUMMARY.md` if Phase 0 passes
