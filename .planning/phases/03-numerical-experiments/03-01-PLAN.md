`generate_synthetic(n_requests, n_vehicles, seed)`:
- Use `random.Random(seed)` (not global random) for reproducibility
- Origins and destinations: uniform random in [0, 20000] × [0, 20000] meters
- 100 meeting points on a 10×10 regular grid: x = i*2000, y = j*2000 for i,j in 0..9
- Request earliest: uniform in [0, 4*3600] (4-hour horizon)
- Request latest: earliest + uniform(5*60, 20*60) (5–20 min window)
- max_ride_time: 45*60 for all requests
- Vehicles: capacity=8, current_position=random in grid, current_time=0

`generate_beijing(n_requests, n_vehicles, seed)`:
- Area: [0, 15000] × [0, 15000] meters
- 80 meeting points: 9×9 grid (81 points), drop last one → 80 points, spacing ~1667m
  Actually use ceil(sqrt(80))=9, so 9×9=81, keep first 80 by row-major order
  x = i * (15000/8), y = j * (15000/8) for i in 0..8, j in 0..8, take first 80
- Request earliest: normal distribution centered at 8*3600 (8am), std=30*60, clipped to [7*3600, 9*3600]
- Request latest: earliest + uniform(5*60, 15*60)
- Origins: random in [0, 15000]², destinations: biased toward center (residential→commercial)
  destination_x = clamp(origin_x + normal(0, 3000), 0, 15000), same for y
- max_ride_time: 40*60
- Vehicles: capacity=8, start at random positions, current_time = 7*3600

Define `SimulationResult` dataclass in experiments/metrics.py (used by all variants):

```python
@dataclass
class PassengerRecord:
    request_id: str
    passenger_type: str       # "price_sensitive" | "time_sensitive" | "walk_sensitive"
    accepted: bool
    wait_time: float          # seconds from request to pickup (0 if rejected)
    pickup_walk: float        # meters
    dropoff_walk: float       # meters
    ivt: float                # in-vehicle time seconds
    direct_time: float        # Euclidean distance / speed (seconds)
    total_disutility: float   # MNL utility value (negative)

@dataclass
class SimulationResult:
    records: list[PassengerRecord]
    total_vehicle_km: float
    cpu_time: float           # seconds

@dataclass
class MetricsResult:
    acceptance_rate: float
    vehicle_km: float
    avg_wait: float
    p95_wait: float
    avg_walk: float
    avg_ivt: float
    detour_ratio: float
    fairness_index: float     # Gini coefficient
    cpu_time: float
```

Implement `compute_metrics(result: SimulationResult) -> MetricsResult`:
- Filter accepted = [r for r in result.records if r.accepted]
- acceptance_rate = len(accepted) / len(result.records) if result.records else 0.0
- vehicle_km = result.total_vehicle_km
- avg_wait = mean(r.wait_time for r in accepted) or 0.0
- p95_wait = np.percentile([r.wait_time for r in accepted], 95) or 0.0
- avg_walk = mean(r.pickup_walk + r.dropoff_walk for r in accepted) or 0.0
- avg_ivt = mean(r.ivt for r in accepted) or 0.0
- detour_ratio = mean(r.ivt / r.direct_time for r in accepted if r.direct_time > 0) or 0.0
- fairness_index: Gini of total_disutility across accepted passengers (use absolute values)
  Gini formula: G = sum_i sum_j |x_i - x_j| / (2 * n^2 * mean(x))
  Return 0.0 if fewer than 2 accepted passengers
- cpu_time = result.cpu_time

Write tests/test_metrics.py covering: empty result, single passenger, known Gini value (equal disutility → Gini=0), p95 with 20 passengers.
