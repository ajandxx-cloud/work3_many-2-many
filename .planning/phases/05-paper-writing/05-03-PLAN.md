\subsection{Heuristic Algorithm: Rolling Horizon ALNS}
~600 words. Describe the full heuristic in 4 sub-parts:

\subsubsection{Candidate Generation}
For each arriving request r, generate top-k^{top} pickup candidates M_r^P (sorted by d(o_r,m)) and top-k^{top} dropoff candidates M_r^D (sorted by d(δ_r,m)), filtered by walking radii ρ^P and ρ^D. Complexity: O(|M| log |M|) per request.

\subsubsection{Online Insertion Evaluation}
For each bundle (m^P, m^D) ∈ M_r^P × M_r^D, enumerate all vehicles v ∈ V and insertion position pairs (π^P, π^D) with π^D > π^P. Apply fast feasibility check (O(1) per position using forward/backward schedule arrays) against constraints con:capacity, con:tw-early, con:tw-late, con:ridetime, con:time-consistency. Select best feasible insertion by incremental cost.

\subsubsection{ALNS Destroy and Repair Operators}
Five operators (from HEUR-05):
1. Request reassignment: remove k randomly selected requests, reinsert greedily
2. Pickup/dropoff pair swap: exchange (m^P, m^D) assignments between two requests
3. Insertion position exchange: swap route positions of two nodes on same vehicle
4. Route segment reconstruction: remove a contiguous route segment, reinsert optimally
5. Worst-removal: remove requests with highest marginal cost, reinsert

Operator selection: roulette-wheel based on historical improvement scores (standard ALNS).

\subsubsection{Rolling Horizon Loop}
Every Δ=5 minutes, trigger ALNS on active set R^{act}(t^{now}) over window [t^{now}, t^{now}+H] with H=30 minutes. Committed nodes (already served) are locked. Average decision time per request: <1 second on instances with 200 requests and 15 vehicles (per HEUR-06 benchmark).

\subsection{Complexity and Scalability}
~200 words. State time complexity of candidate generation O(|M| log |M|), insertion evaluation O(|V|·|σ_v|²·k^{top}²), ALNS per iteration O(n·|V|·|σ_v|). Note that rolling horizon bounds the active set size, keeping per-step complexity manageable. Reference the response-time benchmark result (<1 second).
