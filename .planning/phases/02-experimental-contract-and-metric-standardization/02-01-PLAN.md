# 02-01 Plan: Experiment Contract and Baseline Taxonomy

<objective>
Create the main Phase 2 experiment contract and baseline taxonomy. These documents define which comparisons are behavioral evidence, which are deterministic diagnostics, how conceptual method labels map to current or missing code variants, and what claim boundaries later phases must preserve.
</objective>

## Scope

In scope:

- Define experiment families for unconstrained behavioral evidence, matched-coverage supplementary evidence, fixed accepted-set diagnostics, deterministic feasibility/routing diagnostics, and algorithm-validation diagnostics.
- Define the four-axis baseline taxonomy: service design, passenger response, routing algorithm, diagnostic role.
- Map conceptual methods to current code variants in `experiments/variants.py` and mark missing variants such as `SingleSidedDropoff`.
- Lock claim boundaries that prevent coverage-confounded or mixed-response comparisons from supporting unconditional superiority claims.

Out of scope:

- Running experiments.
- Editing `experiments/*.py`, `src/drt/*.py`, `results/*`, or manuscript prose.
- Final passenger-choice calibration, which belongs to Phase 3.
- Detailed ALNS/MILP exact-model design, which belongs to Phase 4.
