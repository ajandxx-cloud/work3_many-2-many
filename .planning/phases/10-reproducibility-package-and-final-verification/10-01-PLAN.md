# Plan 10-01: Result Manifest and Prerequisite Inventory

<objective>
Create the structured Phase 10 result manifest that inventories every current
artifact by evidence family, marks missing Phase 6 and Phase 8 prerequisites as
hard blockers, and prevents pilot, legacy, or diagnostic outputs from being used
as final claim evidence.
</objective>

<must_haves>
- D-01: Phase 10 must fail closed if Phase 6 formal results/artifacts or Phase 8 claim-gate files are missing.
- D-02: The manifest must still be created as a blocked/pending artifact when prerequisites are missing.
- D-03: Hard blockers are `06_FORMAL_SYNTHETIC_RESULTS.md`, `08_CLAIM_EVIDENCE_MATRIX.md`, `08_SUPPORTED_CLAIMS.md`, and `08_UNSUPPORTED_OR_EXPLORATORY_CLAIMS.md`.
- D-04: Use the exact blocked wording `Blocked: prerequisites missing`.
- D-06: Pilot and legacy artifacts may be listed only as `readiness`, `provenance`, or `legacy`, never as headline evidence.
- D-09: The manifest must be layered by evidence family, then list row-level artifacts.
- D-10: Each artifact row must include `path`, `role`, `evidence_family`, `status`, `source_command`, `inputs`, `outputs`, `code_revision`, `claim_link`, and `notes/blockers`.
- D-11: Code revision and dependency commands are required; checksums and dependency snapshots are recommended improvements, not hard blockers.
- D-12: When Phase 8 is missing, use `pending Phase 8` as `claim_link` and mark artifacts unable to support final claims.
</must_haves>

<truths>
- `.planning/phases/06-formal-synthetic-experiments/06_FORMAL_SYNTHETIC_RESULTS.md` is absent in this workspace.
- `.planning/phases/08-evidence-synthesis-and-claim-gate/` is absent in this workspace.
- `results/pilot/phase05/` exists and contains readiness-only pilot outputs.
- `results/` contains legacy, diagnostic, robustness, and manuscript-adjacent generated outputs that require evidence-role labels before use.
- `manuscript/figures/scripts/` and `manuscript/figures/` exist and must be documented as reproducible display assets, not automatically final evidence.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Missing Phase 6 or Phase 8 evidence is hidden by a mostly complete manifest | high | Add a hard prerequisite section and use `Blocked: prerequisites missing` for missing upstream evidence |
| Pilot results are treated as formal evidence | high | Force pilot rows to `readiness` or `provenance` roles |
| Legacy CSVs or figures support headline claims without Phase 8 | high | Set `claim_link` to `pending Phase 8` and status to `Blocked` or `Not final evidence` |
| Manifest cannot be converted to machine-readable form later | medium | Use stable column names and one artifact per row |
| Dependency or revision provenance is missing | medium | Include `code_revision` and dependency-command fields plus improvement notes for checksums/snapshots |
</threat_model>
