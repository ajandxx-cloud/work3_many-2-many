```python
import math
from drt.types import Request, MeetingPoint

def euclidean(a: tuple[float, float], b: tuple[float, float]) -> float:
    return math.sqrt((a[0]-b[0])**2 + (a[1]-b[1])**2)

def generate_candidates(
    request: Request,
    meeting_points: list[MeetingPoint],
    rho: float,
    k_top: int,
    side: str  # 'pickup' or 'dropoff'
) -> list[MeetingPoint]:
    """
    HEUR-01: Filter meeting points by walking radius rho (ρ^P or ρ^D),
    rank by distance, return top k_top candidates.
    """
    ref = request.origin if side == 'pickup' else request.destination
    within = [(euclidean(ref, mp.coords), mp) for mp in meeting_points
              if euclidean(ref, mp.coords) <= rho]
    within.sort(key=lambda x: x[0])
    return [mp for _, mp in within[:k_top]]
```

```python
def check_feasibility(
    route: Route,
    request: Request,
    pickup_mp: MeetingPoint,
    dropoff_mp: MeetingPoint,
    pos_p: int,   # insertion position for pickup (0-indexed, before existing stop at pos_p)
    pos_d: int,   # insertion position for dropoff (must be > pos_p)
    vehicle: Vehicle,
    travel_speed: float = 1.0  # units/minute, used to compute travel times
) -> tuple[bool, str]:
```

Implementation logic:
1. Precedence check first (cheapest): if pos_d <= pos_p, return (False, "precedence")
2. Build new stop list by inserting pickup_mp at pos_p and dropoff_mp at pos_d
3. Recompute schedule times: start from vehicle.current_time, add Euclidean travel time between consecutive stops (distance / travel_speed)
4. Capacity check: track occupancy (+1 at pickup, -1 at dropoff); if any stop exceeds vehicle.capacity, return (False, "capacity")
5. Time window check: pickup scheduled time < request.earliest → (False, "tw_early"); > request.latest → (False, "tw_late")
6. Ride time check: scheduled_dropoff - scheduled_pickup > request.max_ride_time → (False, "ride_time")
7. Route duration check: last_stop_time - vehicle.current_time > vehicle.max_route_duration → (False, "route_duration")
8. All pass → (True, "")

The route.stops list contains (MeetingPoint, float) tuples where float is the scheduled arrival time. When building the new schedule, recompute all times from scratch using travel_speed (do not trust existing times after insertion point).
