**Destroy operators (HEUR-05):**

1. `random_removal(state, k, rng) -> ALNSState`:
   - Remove k random requests from routes
   - For each removed request: find which vehicle/positions it occupies, remove both pickup and dropoff stops from route.stops
   - Add removed requests to state.unassigned

2. `worst_removal(state, k, vehicles, travel_speed) -> ALNSState`:
   - Compute cost contribution of each assigned request (IVT + walk distance proxy)
   - Remove k requests with highest cost contribution
   - Same stop removal logic as random_removal

3. `related_removal(state, k, rng, travel_speed) -> ALNSState`:
   - Pick a random seed request from assigned requests
   - Score all other assigned requests by spatial+temporal relatedness:
     `relatedness = euclidean(r1.origin, r2.origin) + abs(r1.earliest - r2.earliest)`
   - Remove k most related requests (lowest relatedness score = most similar)

4. `greedy_insertion(state, vehicles, meeting_points, rho_p, rho_d, k_top, cost_weights, travel_speed) -> ALNSState`:
   - For each request in state.unassigned (in random order):
     - Call evaluate_insertion to find best insertion
     - If found: apply insertion to route (insert stops at pos_p, pos_d), remove from unassigned
     - If not found: leave in unassigned (will be rejected)

5. `regret_insertion(state, vehicles, meeting_points, rho_p, rho_d, k_top, cost_weights, travel_speed) -> ALNSState`:
   - Regret-2 heuristic: for each unassigned request, compute best and second-best insertion cost
   - regret_value = second_best_cost - best_cost (higher = more urgent to insert now)
   - Insert the request with highest regret value first
   - Repeat until no more feasible insertions
